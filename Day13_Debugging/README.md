# Day 13: Debugging & Error Handling 🔍

Today, I focused on the art of debugging and making my Python code more robust against unexpected errors.

### 🛠 Key Learning Outcomes
* **Identifying Bug Types:** I practiced finding and fixing common issues such as Syntax Errors, Comparison Errors (using `==` correctly), and Value Errors.
* **Error Handling with Try-Except:** I learned how to use `try` and `except` blocks to prevent programs from crashing when they encounter invalid user input or data.
* **PyCharm Debugger Mastery:** I explored the built-in PyCharm debugger to execute code step-by-step, monitor variable states, and pinpoint logic flaws in real-time.

### 💻 Practice Example: Try-Except
I implemented a small exercise (`try_except_exercise.py`) to demonstrate how to handle potential `ValueError` exceptions when converting strings to integers.

### 🧠 Reflections
Debugging isn't just about fixing a mistake; it's about understanding the flow of the program. Using a professional IDE like PyCharm has significantly improved my ability to trace my code's logic.
